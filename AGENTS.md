## Development Workflow

### Common Tasks

**Creating a feature or fix:**
1. Create a branch from `main` with a descriptive name
2. Implement changes in isolated commits
3. Open a pull request for review before merging to main

### Branch Conventions
- Use lowercase with hyphens: `feature/add-x`, `fix/issue-y`, `docs/update-readme`
- Prefix with type: `feature/`, `fix/`, `docs/`, `chore/`, etc.

### Commit Messages
- Start with a verb: "Add", "Fix", "Update", "Remove"
- Keep first line under 50 characters
- Provide context in the body for complex changes
